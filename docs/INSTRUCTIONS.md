# Sazanami Inc. コーポレートサイト制作メモ

本書は Sazanami Inc.（sazanami.ai）のコーポレートサイト制作に関する要件整理です。Next.js + Tailwind CSS による静的エクスポートを前提とします。

## プロダクト概要

- 会社名: **Sazanami Inc.**
- 提供サービス: **Sazanami-Analytics**（高校生の活動ログを証拠化するAIポートフォリオエンジン）
- 目的: 会社の信頼性を短時間で理解させ、サービス価値を伝達し、問い合わせ・導入・スポンサー連携へ誘導する。

## 想定ターゲット

| ペルソナ         | 主目的                       | 主要訴求                 |
| ---------------- | ---------------------------- | ------------------------ |
| 高校生           | 努力を見える化し提出物に活用 | 成長の可視化・自己効力感 |
| 保護者           | 子の努力と安全性の両立       | 安全なデータ管理         |
| 教員・学校       | 評価業務の効率化・透明化     | 提出書類の統一と再現性   |
| 企業・スポンサー | 若者支援・採用・社会貢献     | 成長証跡の信頼性         |

## ページ構成

1. **トップページ** – ヒーローコピー「高校生の努力を証拠に変える」。価値訴求と信頼証拠、CTAを配置。
2. **サービス紹介** – 課題・解決構造（データ収集→提出様式変換）と成果物例を説明。
3. **会社情報** – ミッション、経営陣、所在地、沿革、連携実績。
4. **ポリシー** – プライバシーポリシー、利用規約、AIポリシー、コンテンツポリシー、セキュリティ声明（ドラフト表記）。
5. **お問い合わせ** – 目的選択式。フォームは静的サイトのためメールクライアント起動方式を採用し、返信SLA（3営業日以内）を明記。

## 提供価値のポイント

- 日常的な活動ログを「証拠」として再利用可能にする。
- AIによる自動整形と提出様式変換。
- 教育現場で求められる安全設計（権限制御、暗号化、監査ログ、削除権）。
- プライバシー保護とデータ利用の透明性。

## 導入モデル

- 個人無料枠（出力回数・保管期間に制限）。
- 学校ライセンス（SSO、監査ログ、教員ダッシュボード）。
- 企業スポンサー（大会や教育プログラムとのタイアップ）。

## 運用・KPI

- 例: 新規訪問→問い合わせ率、問い合わせ→デモ実施率、契約校あたりのアクティブ率、ポートフォリオ更新回数。
- 運用ルール: 月次でトップ更新、四半期ごとに事例追加、インシデント発生時は即時告知。

## 技術的な注意

- Next.js App Router + Tailwind CSS。
- `next.config.ts` に `output: 'export'` を設定して静的配信に対応。
- `npm run build` 後に `next export` を実行すると `out/` に書き出せます。
- Lint/Format コマンド: `npm run lint`, `npm run format`（整形は `npm run format:write`）。
